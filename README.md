【万字文档+源码】基于springboot+vue软件缺陷管理系统-可用于毕设-课程设计-练手学习-学习资料分享-本项目设计并实现了一个面向软件研发团队的缺陷管理系统，旨在解决传统缺陷管理中的信息分散、沟通低效等问题。系统采用前后端分离架构（SpringBoot+Vue2），主要功能包括： 双角色权限管理：区分开发人员和测试人员账号体系，实现权限隔离 全生命周期缺陷跟踪：支持缺陷提交、状态流转（待测试/修复中/已关闭）、图文附件上传等功能 团队协作生态：集成论坛交流和公告推送模块，形成研发配套沟通体系 个人资产管理：统一管理用户信息、发布内容和收藏记录 标准化操作界面：采用Element UI组件库，实现轻量

## 1 项目整体概述

### 1\.1 项目名称

基于软件研发过程中的缺陷管理系统的设计与实现

### 1\.2 项目开发背景

传统软件研发团队在项目迭代过程中，缺陷问题长期依靠微信群、Excel 表格、线下沟通记录，存在诸多管理痛点：

1. 测试人员提交的 Bug 无标准化归档，缺陷归属开发人员、严重等级、修复状态无法统一追踪；

2. 研发与测试人员缺少线上交流渠道，同类缺陷重复提问，沟通成本高；

3. 项目公告、版本更新通知无法统一推送，团队成员信息不同步；

4. 缺陷状态流转全靠人工口头同步，待测试、修复中、已关闭等状态无数字化记录；

5. 员工个人信息、账号权限分散，无法统一维护研发 / 测试两类岗位人员资料。

为规范软件研发全流程缺陷管控，搭建区分**开发人员、测试人员**双角色的缺陷管理系统，实现缺陷全生命周期追踪、团队论坛交流、项目公告推送、个人信息一体化管理，打通测试提 Bug、开发修复、状态复核完整业务闭环。

### 1\.3 项目开发目标与意义

#### 业务实用目标

1. 双角色账号体系：区分开发人员、测试人员独立注册登录，权限隔离；

2. 缺陷全生命周期管理：完整记录缺陷归属人、严重程度、状态、图文描述，支持修改与状态流转；

3. 团队线上交流论坛：支持帖子发布、标题检索、分页浏览、收藏互动；

4. 项目公告资讯推送：多分类公告图文展示、关键词检索，同步项目迭代通知；

5. 个人中心一站式管理：维护头像、姓名、手机号、年龄，支持修改密码、查看我的发布、我的收藏；

6. 轻量化前台页面，统一导航栏布局，操作简单适配研发团队日常办公使用。

#### 技术学习目标

完整落地前后端分离 Web 项目，实现分角色登录、图片上传、多条件检索、分页、收藏、状态流转、轮播图展示等企业级通用功能，可作为计算机专业毕业设计、中小型软件研发内部管理落地项目。

### 1\.4 项目创新亮点

1. **双岗位角色隔离登录**：登录页区分开发 / 测试人员身份单选框，注册入口分两类人员独立注册，权限差异化管控；

2. **缺陷完整业务闭环**：绑定测试提交人、开发修复人，记录缺陷严重等级与状态（待测试 / 修复中 / 已关闭），配套缺陷截图上传；

3. **研发配套交流生态**：内置论坛模块供团队交流技术问题，搭配公告模块同步项目通知，兼顾缺陷管理与团队沟通；

4. **完善个人资产管理**：个人中心整合信息修改、密码重置、个人发布帖子、收藏内容统一查看；

5. **统一轻量化 UI 设计**：全站共用顶部导航栏、轮播 Banner，页面布局标准化，降低团队人员学习成本；

6. **多维度检索功能**：论坛按标题检索、公告按分类 \+ 标题检索、缺陷按名称 / 归属人检索。

## 3 系统角色与权限划分

系统分为两大独立角色，页面操作权限完全隔离：

1. **测试人员**

    - 注册测试账号，登录后可新增缺陷、提交缺陷截图、查看全量缺陷、修改本人提交缺陷；

    - 发布论坛帖子、浏览全部公告、收藏公告 / 帖子；

    - 个人中心维护自身信息，查看本人发布帖子、收藏内容；

    - 可将缺陷状态改为待复测、关闭缺陷。

2. **开发人员**

    - 注册开发账号，登录后查看分配给自己的缺陷，修改缺陷修复状态；

    - 浏览论坛、发布技术交流帖子、查看全量公告；

    - 仅可修改归属自己的缺陷信息，无法新增缺陷；

    - 个人中心维护个人资料、修改密码、管理收藏内容。

## 4 系统需求分析

### 4\.1 功能性需求

1. 账号登录注册模块：区分开发 / 测试身份登录、账号密码校验、分角色注册表单、退出登录；

2. 首页模块：顶部导航、轮播 Banner、系统介绍图文展示；

3. 缺陷信息模块：缺陷列表分页、缺陷详情查看、缺陷图文展示、修改缺陷信息、缺陷状态流转；

4. 论坛模块：帖子标题检索、发布帖子、分页浏览帖子列表、帖子收藏；

5. 公告资讯模块：多分类筛选、标题关键词搜索、公告图文详情展示；

6. 个人中心模块：头像 / 姓名 / 手机号 / 年龄修改、密码重置、我的发布、我的收藏；

7. 通用配套功能：图片上传、分页、关键词检索、收藏 / 取消收藏、轮播图切换。

### 4\.2 非功能性需求

1. 性能需求：页面接口响应≤1\.2s，单表 1000 条数据流畅分页，图片限制 2MB 自动压缩；

2. 安全需求：Token 登录拦截，未登录禁止访问后台页面；表单非空、手机号格式校验；不同角色仅能操作自身权限内数据；

3. 易用性需求：全站统一导航栏，操作按钮布局标准化，弹窗提示操作结果；检索区简化筛选逻辑；

4. 兼容性需求：适配 1366×768 及以上分辨率 PC 端浏览器。

### 4\.3 可行性分析

1. 经济可行性：全部开发工具开源免费，本地部署即可投入团队使用，替代 Excel 线下记录，减少沟通与文档整理工时；

2. 技术可行性：SpringBoot\+Vue 为成熟主流技术，分角色登录、图片上传、分页检索、状态流转均为标准 CRUD 功能，无高并发分布式复杂难点；

3. 操作可行性：页面布局简洁，研发、测试人员仅需基础电脑操作能力即可快速上手缺陷提交、修复、论坛交流；

4. 社会可行性：系统仅用于企业内部软件研发缺陷管控，无违规业务，员工信息本地数据库存储，隐私安全可控。

## 5 系统总体设计

### 5\.1 前后端分离整体三层架构

1. **前端展示层**：Vue 前台页面，渲染登录、首页、缺陷、论坛、公告、个人中心页面，通过 Axios 发送 HTTP 请求调用后端接口，不直接操作数据库；

2. **后端服务层（SpringBoot 四层分层）**

    - Controller 控制层：接收前端请求，校验用户角色身份，调用业务层，返回统一 JSON 结果；

    - Service 业务层：实现缺陷状态流转、帖子收藏、权限校验、数据统计核心逻辑；

    - Mapper 持久层：MyBatis 执行数据库增删改查关联 SQL；

    - Entity 实体层：映射数据库所有数据表字段；
    配套工具包：文件上传工具、Token 权限工具、分页工具、图片压缩工具；

3. **数据持久层（MySQL）**：存储用户、缺陷、论坛帖子、公告、收藏关联数据，使用事务保证缺陷状态修改、帖子发布操作原子性。

### 5\.4 数据库总体 E\-R 设计

核心实体关联关系：

1. 用户 \(user\) 1 对多 缺陷 \(bug\)：一名测试人员可提交多条缺陷，一名开发人员可绑定多条待修复缺陷；

2. 用户 \(user\) 1 对多 论坛帖子 \(forum\)：一名用户可发布多条交流帖子；

3. 用户 \(user\) 1 对多 收藏 \(collect\)：用户可收藏公告、论坛帖子；

4. 用户 \(user\) 1 对多 公告 \(notice\)：管理员发布多条项目公告，普通用户仅浏览。

### 5\.5 核心数据表结构说明

1. `user` 用户表：区分角色（开发 / 测试）、账号、密码、姓名、性别、年龄、手机号、头像、注册时间；

2. `bug` 缺陷表：缺陷名称、测试人 ID、开发人 ID、严重程度、缺陷状态、缺陷描述、缺陷截图、提交时间；

3. `forum` 论坛帖子表：帖子标题、发布人 ID、帖子内容、发布时间；

4. `notice` 公告资讯表：公告标题、分类名称、公告配图、公告正文、发布时间；

5. `collect` 收藏关联表：用户 ID、资源类型（帖子 / 公告）、资源 ID、收藏时间。

## 7 核心业务流程设计

### 7\.1 用户注册登录流程

1. 新用户打开登录页，选择对应身份（开发 / 测试），点击对应注册按钮；

2. 填写完整注册表单、上传头像，后端校验账号唯一性，写入 user 表并标记角色；

3. 返回登录页，输入账号密码 \+ 选择对应角色，后端校验账号密码与身份匹配；

4. 校验成功生成 Token 存入前端 LocalStorage，跳转系统首页；

5. 未携带 Token 访问任意页面时，拦截器自动跳转登录页面；右上角「退出登录」清空 Token，返回登录页。

### 7\.2 缺陷提交、流转、修改流程

1. 测试人员登录，进入缺陷信息模块点击添加，填写缺陷名称、选择归属开发人员、录入严重等级、上传缺陷截图、填写缺陷描述，提交生成缺陷，默认状态「待测试」；

2. 开发人员登录，查看分配给自己的缺陷，点击修改将状态改为「修复中」；

3. 测试人员复测后，可修改缺陷状态为「已关闭」；

4. 任意角色进入缺陷详情页，可查看完整缺陷图文信息，仅提交测试人、归属开发人拥有修改权限。

### 7\.3 论坛帖子发布、检索、收藏流程

1. 用户进入论坛页面，点击「发布帖子」，填写标题与正文提交，存入 forum 表；

2. 在搜索框输入标题关键词，点击查询，页面过滤匹配帖子分页展示；

3. 点击帖子详情页收藏按钮，新增收藏记录至 collect 表；再次点击取消收藏，删除对应记录；

4. 用户进入个人中心「我的发布」，查看本人所有发帖记录。

### 7\.4 个人信息维护、密码修改流程

1. 右上角点击「个人中心」进入页面，可直接修改姓名、性别、年龄、更换头像，提交更新 user 表；

2. 点击右侧「修改密码」，输入原密码、新密码、确认密码，校验原密码正确后更新数据库密码字段；

3. 点击「我的收藏」，分页展示所有收藏的公告、论坛资源。

## 8 系统测试方案

### 8\.1 测试类型与方法

采用**黑盒功能测试**，分别模拟开发人员、测试人员两类角色完整操作流程，不查看底层代码，验证页面交互、接口逻辑、数据库存储、角色权限是否符合需求；
测试覆盖模块：登录注册、缺陷管理、论坛、公告、个人中心、收藏、分页检索。

### 8\.3 测试结果总结

系统分角色权限拦截、登录校验、缺陷全生命周期流转、论坛公告检索收藏、个人信息修改功能均可稳定运行；表单格式校验、图片上传、分页、轮播图无异常；不同角色数据隔离有效，无越权操作漏洞，完全满足软件研发团队缺陷管控、线上交流业务需求。

## 9 项目部署运行完整步骤

### 9\.1 环境准备

1. 安装 JDK1\.8 并配置 JAVA\_HOME 环境变量；

2. 安装 MySQL8\.0，创建本地 root 账号，开放 3306 端口；

3. 安装 Node\.js v16，配置 npm 包管理工具；

4. IDEA 导入后端项目，VS Code 打开前端文件夹。

### 9\.2 数据库初始化

1. Navicat 新建数据库 `bug_manage`，字符集 utf8mb4；

2. 执行项目 sql 文件夹内完整建表脚本，生成用户、缺陷、论坛、公告、收藏全部数据表；

3. 可预先插入测试开发账号、测试账号用于调试。

### 9\.3 SpringBoot 后端配置启动

1. 打开`application.yml`配置文件，修改 MySQL 数据库用户名、密码为本地数据库账号；

2. 配置文件上传存储路径、后端服务端口（默认 8080）；

3. Maven 刷新全部依赖包，等待依赖下载完成；

4. 运行启动类`BugManageApplication`，控制台打印启动成功日志，后端接口地址：`http://localhost:8080`。

### 9\.4 Vue 前端项目启动

1. VS Code 进入前端`bug-manage-front`目录，终端执行`npm install`安装依赖；

2. 修改 api 全局请求地址，指向本地后端`http://localhost:8080`；

3. 执行`npm run serve`启动前端页面；

4. 浏览器访问前端本地地址，选择对应角色账号密码登录进入系统。
